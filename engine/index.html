The "Instruction vs. Content" Paradox

A major breakthrough during development was realizing that LLMs do not distinguish cleanly between "instructions about content" and "content itself" unless the prompt architecture makes the separation explicit. This manifested as the 'AI Echo' issue, where the 31B model would frequently leak its internal instructions (e.g., "OBJECTIVE: Write a deep-dive") into the final prose.

Failed Fixes (And Why They Failed)

These are the common prompt-engineering tactics we tried first, which proved entirely insufficient for complex, instruction-dense workflows:

  • Lowering Temperature: Did not stop the echo; it just made the echoed instructions rigidly deterministic and repetitive.
  • Increasing Temperature: Broke the rigid echo loops, but caused the model to hallucinate technical facts and deviate wildly from the blueprint.
  • Stronger System Prompts: Yelling "DO NOT WRITE OUTLINES" inside the system prompt failed. Negative constraints are notoriously weak in large LLMs when the context window is flooded with structural examples.
  • Regex Cleaning After Generation: Brittle and unscalable. Headers like [PART 1] mutate unpredictably, making regex matching a game of whack-a-mole.
  • Chain-of-Thought (CoT) Suppression: Disabling reasoning made the model write faster, but it lost the ability to synthesize the SEO gap data intelligently.

The Actual Fixes (The Gold Standard)

To achieve production-grade stability, we abandoned "prompt tweaking" and moved to Architectural Isolation:

  • Reframing Blueprint → Source Data: We wrapped the planning output in [SOURCE DATA — DO NOT REPRODUCE]. This mentally decoupled "what to do" from "what to know" for the model.
  • Prose-Prefill Anchoring: By pre-supplying the first sentence of the article (e.g., # Title \n\n Deploying EVSE...) into the model's response turn, we physically forced the model into a "continuation" state, bypassing the instruction-echo phase entirely.
  • Sanitizing Poison Headers: We stripped metadata tags (like ROLE: or META-PROMPT) before feeding the blueprint to the writer to remove structural temptation.
  • Separating Knowledge from Task: The task instruction ("Write a dense technical prose...") was explicitly moved to the bottom of the prompt, acting as the final binding constraint after the source data.
  • Rescue Generation: If an echo is mathematically detected in the output stream, the system intercepts it and triggers a high-temperature "Rescue Prompt" using heavily compressed requirements.
